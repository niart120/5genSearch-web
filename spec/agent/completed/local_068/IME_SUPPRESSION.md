# 数値・16進入力での IME 自動無効化 仕様書

**ステータス: 凍結 (対応断念)**

## 凍結理由

16進数入力フィールドで IME を自動無効化する実用的な方法が存在しない。

調査した手段:

| 方式 | 結果 | 理由 |
|------|------|------|
| `ime-mode: inactive` (CSS) | 不可 | 非標準。Chrome 未実装、Firefox v98 で削除済み |
| `inputMode="numeric"` | 不適 | 0-9 のみ。モバイルで A-F が入力不可 |
| `type="url"` | 不適 | IME 無効化は仕様外の副作用 (ブラウザ実装依存)。`:invalid` 疑似クラス適用、モバイルで URL キーボード表示、スクリーンリーダーが「URL 入力」と読み上げる |
| `type="email"` | 不適 | 同上 |
| composition イベントガード | 可だが限定的 | IME 自体は無効化されず、composition 中の中間テキスト表示は残る。既存の `filterHex` + blur 正規化で実害は軽微 |

現状の `filterHex` (非16進文字除去) と `onBlur` (値の正規化) により、IME 経由の入力でも最終的な値は正しくフィルタされている。composition 中の一時的な表示ノイズは残るが、実害は軽微と判断し対応を見送る。
